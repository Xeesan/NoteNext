# NoteNext Website

Marketing & SEO site for the [NoteNext](https://github.com/suvojeet-sengupta/NoteNext) Android app. Built with **Astro 5** + **Tailwind 4**, deploys as a static site to **Cloudflare Pages** at `notenext.suvojeetsengupta.in`.

## Stack

- **Astro 5** — zero-JS by default, ships pure HTML for marketing pages
- **Tailwind 4** (via `@tailwindcss/vite`)
- **`@astrojs/sitemap`** — auto-generated `sitemap-index.xml`
- Static output → ~95–100 Lighthouse, no Cloudflare Functions cost

## Local development

```bash
cd F:\NoteNext\website
npm install
npm run dev      # http://localhost:4321
npm run build    # outputs ./dist
npm run preview  # serves the built site locally
```

The changelog page reads `../changelog.json` at build time, so it stays in sync with the app's release log.

## When the app ships to Play Store

Edit `src/consts.ts`:

```ts
playStoreUrl: 'https://play.google.com/store/apps/details?id=com.suvojeetsengupta.notenext',
```

The download buttons across the site flip to "Get on Google Play" automatically.

## Deploying to Cloudflare Pages

You already deploy `suvojeetsengupta.in` via Cloudflare Pages. We'll add a **second Pages project** for the subdomain.

### Option A — Pages connected to GitHub (recommended)

1. **Push the `website/` folder to GitHub.** Either:
   - Commit it inside the existing `NoteNext` repo (recommended — it's already there), or
   - Create a separate `notenext-website` repo

2. **Cloudflare dashboard → Workers & Pages → Create → Pages → Connect to Git.**

3. Select the `NoteNext` repo, branch `main`. Configure build:

   | Setting | Value |
   |---|---|
   | Framework preset | **Astro** |
   | Build command | `npm install && npm run build` |
   | Build output directory | `dist` |
   | Root directory | `website` |
   | Node version (env var `NODE_VERSION`) | `20` |

4. **Save and Deploy.** First build takes ~1–2 minutes. You'll get a `*.pages.dev` URL.

5. **Custom domain → Set up a custom domain → `notenext.suvojeetsengupta.in`.** Cloudflare will create the CNAME for you because the parent zone is already on Cloudflare. SSL auto-provisions in ~30 seconds.

### Option B — Deploy from local with Wrangler

```bash
npm install -g wrangler
cd F:\NoteNext\website
wrangler login
npm run build
wrangler pages deploy dist --project-name notenext-website
```

Then set the custom domain in the dashboard as in Option A step 5.

## DNS — connecting `notenext.suvojeetsengupta.in`

If your `suvojeetsengupta.in` zone is on Cloudflare, you don't have to do anything manual — adding the custom domain in the Pages project automatically creates a `CNAME notenext → notenext-website.pages.dev` (proxied, orange cloud) in your zone.

If you want to verify or do it manually:

1. **Cloudflare → DNS → Records → Add record**
2. Type `CNAME`, Name `notenext`, Target `notenext-website.pages.dev`, Proxy **on**, TTL Auto.

## SEO checklist (already wired)

- ✅ Per-page `<title>`, `<meta description>`, canonical URLs
- ✅ Open Graph + Twitter cards (image: `public/og-image.png`)
- ✅ JSON-LD `MobileApplication` schema on home page (rich Google result with rating, price, screenshots)
- ✅ JSON-LD `WebSite` schema site-wide
- ✅ JSON-LD `BreadcrumbList` on inner pages
- ✅ JSON-LD `FAQPage` on `/faq`
- ✅ `sitemap-index.xml` auto-generated by `@astrojs/sitemap`
- ✅ `robots.txt` allowing all + sitemap link
- ✅ Semantic HTML, alt-text-ready images, skip-to-content link
- ✅ Strict CSP, HSTS, X-Content-Type-Options, Referrer-Policy via `_headers`
- ✅ Mobile-first responsive layout
- ✅ Theme color, web manifest

### Post-launch SEO tasks

1. **Submit to Google Search Console:**
   - Add property `https://notenext.suvojeetsengupta.in`
   - Verify via DNS TXT record (Cloudflare → DNS → Add TXT)
   - Submit `sitemap-index.xml`
2. **Submit to Bing Webmaster Tools.** Same flow.
3. **PNG OG image:** Generated `public/og-image.png` (1200×630) from the SVG source for universal compatibility (older Facebook clients, Slack 'unfurls', etc.). The configuration in `src/consts.ts` is updated to use `/og-image.png`.
4. **Add real screenshots** to `public/screenshots/` once available — the home page hero currently uses a CSS mockup as a placeholder.
